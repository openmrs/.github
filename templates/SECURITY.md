# Security Policy
 
## Reporting a Vulnerability
 
Please report security issues privately. Don't open a public issue, pull request, forum thread, or Slack message.
 
- Preferred: open a private vulnerability report on this repo (**Security** tab > **Report a vulnerability**). This lets us use GitHub's advisory and CVE tooling.
- Or email <security@openmrs.org>.

Please include as much detail as possible: the steps to reproduce, the affected versions, the expected vs. actual results, and anything else that helps us react faster. We prefer a text write-up with a proof-of-concept over screenshots or videos.
 
Please don't test against any system holding real patient data.
 
For our full vulnerability management process, severity targets, and disclosure steps, see [om.rs/security101](http://om.rs/security101).
